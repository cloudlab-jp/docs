# Roadmap

CloudLab JP is a long-term open-source initiative focused on building a comprehensive cloud engineering ecosystem.

Rather than growing through isolated repositories, the organization follows a structured roadmap where each project strengthens the ecosystem as a whole.

The roadmap below outlines the major milestones planned for CloudLab JP.

---

# Phase 1 — Foundation

**Status:** ✅ Completed

Establish the organizational foundation.

## Goals

- Create the GitHub organization.
- Define the visual identity and branding.
- Publish community standards.
- Create engineering templates.
- Establish documentation standards.
- Define repository conventions.

## Deliverables

- Branding repository
- Organization profile
- Community health files
- Engineering template
- Documentation platform

---

# Phase 2 — Documentation

**Status:** 🚧 In Progress

Build a comprehensive knowledge base covering modern cloud engineering topics.

## Areas

- AWS
- Terraform
- Docker
- Kubernetes
- GitHub
- Platform Engineering
- Cloud Architecture
- DevOps
- Security
- Certifications

## Goals

- Publish learning paths.
- Create architecture references.
- Build reusable examples.
- Maintain production-inspired documentation.

---

# Phase 3 — Infrastructure

**Status:** 📅 Planned

Develop reusable Infrastructure as Code modules and deployment examples.

## Planned Technologies

- Terraform
- AWS CloudFormation
- Docker
- GitHub Actions

## Objectives

- Reusable infrastructure modules.
- Secure cloud patterns.
- Reference architectures.
- Deployment automation.

---

# Phase 4 — Labs

**Status:** 📅 Planned

Create practical laboratories that reinforce the documentation through hands-on experience.

## Planned Content

- AWS labs
- Terraform labs
- Docker labs
- Kubernetes labs
- CI/CD exercises
- Platform engineering challenges

Each lab will be designed to simulate real engineering scenarios.

---

# Phase 5 — Open Source Projects

**Status:** 📅 Planned

Publish standalone open-source projects demonstrating engineering best practices.

Examples include:

- Automation tools
- Internal developer platforms
- Cloud utilities
- Developer productivity tools
- Infrastructure tooling

---

# Phase 6 — Learning Paths

**Status:** 📅 Planned

Provide structured learning journeys that combine documentation, labs, and projects.

Example learning paths include:

- AWS Cloud Practitioner
- AWS Solutions Architect Associate
- Terraform Fundamentals
- Docker Essentials
- Kubernetes Fundamentals
- GitHub Actions
- Platform Engineering Foundations

Each learning path will guide readers from introductory concepts to practical implementations.

---

# Phase 7 — Community

**Status:** 📅 Planned

Foster an open and collaborative engineering community.

Future initiatives may include:

- Community contributions
- Technical discussions
- Architecture reviews
- Open-source collaboration
- Educational content

---

# Long-Term Vision

CloudLab JP aims to become a trusted engineering resource for developers, cloud engineers, and platform engineers.

By combining documentation, reusable templates, practical laboratories, and open-source projects, the organization seeks to provide a cohesive learning experience grounded in real-world engineering practices.

The roadmap will continue evolving as the community grows and new technologies emerge.
